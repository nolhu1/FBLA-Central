# Onboarding and Profile Redesign

This redesign turns onboarding and profile into one shared member identity system instead of a generic account form followed by a plain settings screen.

## What changed

- Rebuilt onboarding into a step-based flow with focused screens for welcome, chapter identity, member basics, interests, alerts, and completion.
- Added shared profile/onboarding components for progress, chapter selection, interest chips, notification cards, identity headers, metric tiles, and settings entry rows.
- Expanded the onboarding payload to include notification preferences so personalization setup affects more than just names and interests.
- Rebuilt the main profile screen around FBLA identity, chapter/state context, momentum, saved-item shortcuts, interests, and clear settings entry points.
- Added a dedicated edit-profile route that reuses the same member setup model for identity, interests, and alert preferences.
- Reworked the preferences screen into a cleaner review surface for notification, privacy, and accessibility settings.

## Product outcome

The member experience now feels much more intentional and FBLA-specific. Onboarding clearly explains why setup matters, and profile now acts as a real identity and personalization hub instead of a generic settings page.
